# ocwt v0.1.0

Initial Python release of `ocwt`.

## What this release includes

- Rebuild of `ocwt` from the Bash script into a typed Python package (`src/` layout).
- CLI commands implemented for:
  - `open`
  - `close`
  - `completion`
  - `config`
- Worktree lifecycle support:
  - create or reuse worktrees
  - safe branch naming and fallback behavior
  - close with protected branch safeguards (`main`/`master`/base)
- Symlink management parity for linked worktrees:
  - `.opencode`
  - `.idea`
  - untracked `.env` / `.env.*`
- Config subsystem at `~/.config/ocwt/config.json` with command-based management.
- Planning mode support via `ocwt open --plan`:
  - streams one-shot planning output
  - attempts same-session handoff
  - fallback to `--continue` when session id is not available
- Editor integration:
  - configurable editor
  - optional auto-open behavior on worktree open/reuse

## Quality and tooling baseline

- Python 3.12+
- `uv` for environment/dependencies
- `ruff` for lint/format
- `ty` as the only type checker
- `pytest` test suite (unit + integration)

## Notes

- This is the first semantic version release (`v0.1.0`).
- Release workflow expects semantic tags in the form `vX.Y.Z`.
