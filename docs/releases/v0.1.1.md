# ocwt v0.1.1

Patch release to complete the first PyPI publication flow with semantic version tags.

## What this release does

- Publishes `ocwt` using the GitHub release workflow with semantic tags (`vX.Y.Z`).
- Keeps CI gates in release flow (`ruff`, `ty`, `pytest`) before packaging.
- Builds and uploads Python artifacts (`sdist` + wheel) to GitHub Releases.
- Publishes artifacts to PyPI via trusted publishing.

## Included functionality

This release ships the initial Python implementation of `ocwt`, including:

- Typed Python package with `src/` layout and CLI entrypoint.
- `open`, `close`, `completion`, and `config` commands.
- Git worktree create/reuse/close behavior with base-branch protections.
- Shell completion support for bash and zsh, including `@file` completion behavior.
- Config management backed by `~/.config/ocwt/config.json`.
- Planning mode (`ocwt open --plan`) with streamed output and session handoff fallback.
- Editor auto-open integration via config/flags.
- Unit + integration test coverage for key worktree flows.
